# Contributing to Socrates

Socrates is an experimental language-resonance engine. Contributions should move it toward generic discovery, data-driven knowledge, and inspectable evidence.

## What Helps

Useful contributions:

- generic form generation
- similarity algorithms
- activation graph logic
- data loaders and validators
- tests that prevent hardcoded shortcuts
- evidence-first rendering
- clear documentation of confidence and uncertainty

## What To Avoid

Avoid:

- adding semantic keyword maps to Go code
- adding word-specific branches
- making examples pass through hardcoded behavior
- hiding unclear logic behind spiritual language
- claiming resonance proves etymology or truth
- adding hardware, UI, or AI before the core engine works

## Development Principles

- Keep code generic.
- Keep knowledge in data.
- Make every score explainable.
- Preserve uncertainty.
- Label speculation.
- Prefer tests that protect architecture over tests that only check examples.

## Contribution Process

1. Read `TODO.md`.
2. Work on the next unchecked item.
3. Update `TODO.md` progress markers.
4. Add or update tests that prevent regressions.
5. Report files changed, tests run, and known limitations.

## Project Tone

The project can be unusual without becoming inflated.

Use language that clarifies the system. If a sentence sounds profound but does not explain evidence or behavior, rewrite it.
